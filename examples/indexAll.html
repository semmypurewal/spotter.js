Here's a few examples that use spotter. Most of them use the JQuery library in addition to Spotter.

  • The Bieber Example: This example alerts the user whenever there's a new tweet about Justin Bieber. (Note that this application will not show *all* tweets about Justin Bieber, just a sample). The entire program (with the exception of Spotter) is in the source document.
  • The Trends Example: In this example, the spotter notifies the application when the trending topics on twitter change, and the page is updated accordingly.
  • The Gadget Example: This is example is a true mashup; it uses two spotters -- one for twitter trends and one for twitter search. It produces real-time twitter search results based on the tenth trending topic. One spotter notifies the application when the tenth trending topic changes, and the other notifies the application when there are new tweets about that topic. The JavaScript code can be found here.
  • The Identica Example: This example does a real-time search on identi.ca for new data relating to some topic.
  • The Delicious Example: This is example uses a spotter to watch recently bookmarked links on delicious. Unlike the previous gadget apps, it doesn't automatically display the updates -- it buffers them until the user refreshes. The JavaScript code be found here.
  • The Flickr Slideshow Example: This example uses the Flickr API to produce a slideshow from a given search term. Spotter does not act as a daemon in this example. The entire program is in the html document.
  • The Twitpic Slideshow Example: This example uses the twitter API to produce a slideshow from a given search term using images posted on Twitpic rather than images posted on Twitter. The entire program is in the html document.
  • The Facebook Example: This example uses the facebook API to find statuses matching a search term.
  • The Tweetphoto Slideshow Example: CURRENTLY BROKEN Tweetphoto slideshow.

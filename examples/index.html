Here's a few examples that use spotter. Most of them use the JQuery library in addition to Spotter.

  • The Bieber Example: This example alerts the user whenever there's a new tweet about Justin Bieber (at most once every 10 seconds). The entire program (with the exception of Spotter) is in the source document.
  • The Trends Example: In this example, Twitter's trending topics are monitored and the page is updated as necessary. The Spotter checks with the Twitter API every 30 seconds.
  • The Gadget Example: This is example is slightly more interesting; it mimics the ubiquitous Twitter gadget. The current search term is 'oil spill.' The JavaScript code can be found here.
  • The Delicious Example: This is example kinda sucks. It uses the currently buggy delicious module to print out some recent things people have bokmarked. In reality it's a copy/hack of the previous gadget app which is much cooler. The JavaScript code be found here.
  • The Slideshow Example: This example uses the Flickr API to produce a slideshow from a given search term. Spotter does not act as a daemon in this example. The entire program is in the html document.
